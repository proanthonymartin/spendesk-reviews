Spendesk Reviews — Étude de cas

Scraping & analytics de 225 avis Capterra

Voir le dashboard Code source

Contexte

Étude de cas réalisée dans le cadre d'un entretien pour un poste GTM Outbound chez Skello (SaaS de planification RH pour les CHR). L'objectif : démontrer ma capacité à construire une machine d'intelligence concurrentielle — scraper, analyser et présenter des données exploitables.

Choix de la cible : Spendesk, concurrent indirect de Skello sur le segment des outils financiers pour PME/ETI.

Défi technique

Capterra protège ses pages de reviews avec Cloudflare. Les approches classiques (Playwright headless, puppeteer-extra, requests) sont toutes bloquées — réponse 403 ou challenge JS non résolu.

Solution : Chrome CDP

Connexion à une instance Chrome réelle via Chrome DevTools Protocol (--remote-debugging-port=9222). Le navigateur étant un vrai Chrome lancé manuellement, Cloudflare lui fait entièrement confiance — plus de blocage.

chrome.exe --remote-debugging-port=9222
python live_scraper.py  # Playwright → CDP → navigateur de confiance

Pipeline

  1. Navigation pages 1–10 de capterra.com/p/157515/Spendesk/reviews/
  2. Extraction : note, titre, date, auteur, rôle, secteur, avis/contre, corps
  3. Déduplication par (titre, auteur, date) → 225 avis uniques
  4. Export en JSON → alimente le dashboard

Les frustrations principales des utilisateurs Spendesk — échecs de paiement, fonctionnalités manquantes, intégration comptable — sont exactement les angles où un concurrent comme Skello peut se différencier.

Stack technique

Pourquoi ce projet est pertinent pour Skello

  • Scraping réel — contournement de Cloudflare, pipeline de données fonctionnel
  • Analyse concurrentielle — transformation de données brutes en insights métier
  • Stack outbound — mêmes compétences (Python, scraping, automatisation) que pour une machine de prospection
  • Delivery — du concept au déploiement en une session de travail
